Monorepo Dependency Architecture

💡 Click any box with a generated DI design to open its design.html (what the AI sees inside that project).

Legend — fill = framework (libType), border = role

angular: Angular front-end
react: React front-end
browser: browser front-end base env
express: Express / server side
node: node server base env
server: runnable server app (double border)
client: client app, e.g. angular (dashed border)
designed-lib: library with a generated @ApiImplementation design (bold border)
lib: plain library, no generated design (thin border)
Library nodes show their dependency level (L#), their framework env set (e.g. [browser, node]), and their role, and are laid out by level, with the deepest libraries at the bottom. Server and client apps are roots (nothing depends on them), so they are all pinned to the single top row regardless of dependency depth — their labels omit the L#. Transitive dependencies are allowed but not shown.
